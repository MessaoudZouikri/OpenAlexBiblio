# Visualization Guide

The pipeline produces two distinct categories of visual output. Understanding
the difference is important because they are generated by different agents and
require different tools to view.

---

## Two categories of output

| Category | Generated by | Location | Tool to open |
|---|---|---|---|
| **Statistical figures** (PNG + HTML) | `visualization.py` | `data/outputs/figures/` | any browser or image viewer |
| **Network graphs** (GraphML) | `network_analysis.py` | `data/outputs/networks/` | VOSviewer or Gephi |

These two categories are **independent**. Re-running visualization does not
touch the network files, and re-running network analysis does not touch the PNG
figures (though it does update the metrics JSON that `cross_domain_heatmap.png`
reads from).

---

## Statistical figures

All files land in `data/outputs/figures/`.

| File | What it shows |
|---|---|
| `publication_trends.png` | Annual publication counts + domain breakdown by year |
| `citation_distribution.png` | Citation percentile bars, h-index and g-index markers |
| `top_authors.png` | Top 15 authors by paper count and by total citations |
| `domain_distribution.png` | Pie chart of domain shares + top subcategories |
| `concept_landscape.png` | Top 30 OpenAlex concepts by frequency |
| `publication_types.png` | Pareto chart of publication types + frequency table |
| `publication_types_by_domain.png` | Grouped bar chart of types × domain + cross-tab table |
| `cross_domain_heatmap.png` | Raw bibliographic coupling counts between the 4 domains |
| `cross_domain_heatmap_enhanced.png` | 4-panel heatmap: AS, CSI, Jaccard similarity, raw counts |

Reports in `data/outputs/reports/`:

| File | What it shows |
|---|---|
| `report.html` | All figures embedded in a self-contained HTML page |
| `report.md` | Markdown version of the same report |

**To regenerate figures without re-running the full pipeline:**

```bash
python src/agents/visualization.py --config config/config.yaml
```

This reads from already-computed parquet/JSON files in `data/processed/` and
overwrites every PNG and report. It runs in ~10 seconds.

## Network graphs (GraphML)

Network files land in `data/outputs/networks/`. They are **not images** — they
are graph data files containing nodes, edges, and computed metrics. You need
VOSviewer or Gephi to explore them visually.

### Main networks

| File | Network type | Notes |
|---|---|---|
| `bibcoupling_network_raw.graphml` | Bibliographic coupling — all edges | Large; start with the VOS-filtered version |
| `bibcoupling_network_vos.graphml` | Bibliographic coupling — filtered | Edges pruned by association-strength threshold |
| `cocitation_network_raw.graphml` | Co-citation — all edges | Papers that share citing documents |
| `cocitation_network_vos.graphml` | Co-citation — filtered | Recommended for visualization |
| `coauthorship_network.graphml` | Co-authorship | Authors linked by shared papers |
| `keyword_cooccurrence_network_vos.graphml` | Keyword co-occurrence | Concepts appearing together in the same paper |

The `_raw` files contain every edge above the minimum shared-reference floor.
The `_vos` files apply an additional **association-strength threshold** (VOSviewer
convention) that removes weak edges and produces cleaner cluster maps. **Use the
`_vos` variants for publication-quality visualizations.**

### Subfield networks (generated with `--subfield_analysis`)

When the pipeline is run with `--subfield_analysis`, additional per-subcategory
networks are produced:

```
bibcoupling_radical_right_vos.graphml
bibcoupling_european_politics_vos.graphml
bibcoupling_political_theory_vos.graphml
bibcoupling_media_communication_vos.graphml
bibcoupling_interdisciplinary_vos.graphml
cocitation_radical_right_vos.graphml
...
```

### To regenerate network files

Network files are only written by `network_analysis.py`. Re-running
`visualization.py` alone does **not** update them.

```bash
# Via orchestrator — resets and re-runs network_analysis + visualization
python src/agents/orchestrator.py --config config/config.yaml --from-step network_analysis

# Directly (fastest, does not update the checkpoint)
python src/agents/network_analysis.py --config config/config.yaml
```

## Opening networks in VOSviewer

1. Download VOSviewer free from [vosviewer.com](https://www.vosviewer.com)
2. **File → Create → Create a map based on network data**
3. Choose **GraphML format** and select one of the `_vos.graphml` files
4. Set **edge weight field** to `weight`
5. Use **Cluster colors** → the `domain` node attribute is pre-assigned

Start with `bibcoupling_network_vos.graphml` — it gives the clearest cluster
structure for the populism literature.

## Opening networks in Gephi

1. Download Gephi free from [gephi.org](https://gephi.org)
2. **File → Open** → select any `.graphml` file
3. Run **Modularity** (Statistics panel) to detect communities
4. Apply **Force Atlas 2** layout for a readable map
5. Color nodes by the `domain` attribute

## Customising figures

All PNG visualization code is in `src/agents/visualization.py`. Each figure is
a single `fig_*` function. Key parameters:

- **DPI** — default 150 for screen; set to 300 inside the `savefig` call for
  print-ready export
- **Colour palette** — the `colors` list at the top of each `fig_*` function
- **Figure size** — the `figsize` tuple in `plt.subplots(...)`

To add a new figure:

1. Add a `fig_my_plot(data, fig_dir)` function in `visualization.py`
2. Call it from `main()` after the existing calls
3. It will appear in `report.html` automatically on the next run

---

## Adding a custom figure

```python
def fig_my_plot(df: pd.DataFrame, fig_dir: str) -> None:
    fig, ax = plt.subplots(figsize=(10, 6))
    # ... your plot code ...
    plt.tight_layout()
    plt.savefig(f"{fig_dir}/my_plot.png", dpi=150, bbox_inches="tight")
    plt.close()
```

Then in `main()`:

```python
try:
    fig_my_plot(df_classified, fig_dir)
    logger.info("Generated: my_plot.png")
except Exception as e:
    logger.warning("my_plot: %s", e)
```
